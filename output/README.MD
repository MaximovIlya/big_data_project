# output/

This directory stores all pipeline outputs generated by Stages II, III, and IV.

## Contents

```
output/
├── eda/                          # Stage II: EDA insight CSV files
│   ├── insight1_top_categories/  # Top 10 incident categories
│   ├── insight2_by_hour/         # Incidents by hour of day
│   ├── insight3_by_day_of_week/  # Incidents by day of week
│   ├── insight4_by_district/     # Incidents by police district
│   ├── insight5_monthly_trend/   # Monthly trend over years
│   ├── insight6_resolution_rate/ # Resolution rate by category
│   ├── insight7_top_neighborhoods/ # Top 15 neighborhoods
│   ├── insight8_hour_day_heatmap/  # Hour × Day heatmap
│   └── plot_*.png                # Matplotlib visualizations (local notebooks)
├── metrics/
│   └── metrics_summary.json      # Stage III: accuracy + F1 for all 3 models
├── predictions/
│   ├── random_forest/            # Stage III: 5 sample predictions (CSV)
│   ├── linear_svc/
│   └── naive_bayes/
├── sf_incidents_eda.db           # Stage IV: SQLite DB for Superset
└── superset_export/
    └── sf_incidents_dashboard.zip  # Stage IV: Superset v1 export ZIP
```

All files in this directory (except `plot_*.png`) are generated during pipeline execution and are excluded from git.

## EDA Insights

Each `insight*/` subdirectory is a Spark CSV output directory containing a single `part-*.csv` file with a header row. These are written by `scripts/stage2_eda.py` on YARN and copied to local disk by `scripts/stage2.sh` via `hdfs dfs -get`.

## Metrics Format

`metrics/metrics_summary.json` structure:

```json

  },
  "linear_svc": {

    "accuracy": 0.3276,
    "weighted_f1": 0.2248
  },
  "config": {
    "top_n_categories": 10,

    "test_ratio": 0.3
  }
}
```

## Superset Dashboard

`superset_export/sf_incidents_dashboard.zip` is a Superset v1 export importable via:

**Superset UI → Dashboards → (+) → Import dashboard**

It includes:
- SQLite database connection to `sf_incidents_eda.db`
- 8 dataset definitions (one per EDA insight)
- 8 chart definitions (bar, line, heatmap)
- 1 dashboard layout (2-column grid)
